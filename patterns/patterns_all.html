Sliding Window Pattern

Sliding Window

📖 Concept

The Sliding Window pattern is used to solve problems involving subarrays or substrings of a fixed or variable size. Instead of recalculating everything for each window, we efficiently slide the window by adding one element and removing another.

Key Idea: Maintain a window that slides through the array/string, updating the window state incrementally rather than recalculating from scratch.

🎯 Techniques to Master

  • Fixed Size Window: Window size is constant (e.g., find max sum of k consecutive elements)
  • Variable Size Window: Window size changes based on conditions (e.g., smallest subarray with sum ≥ target)
  • Two Pointers: Use left and right pointers to expand/contract the window
  • Hash Map Tracking: Use hash maps to track character/element frequencies in the window
  • Window State: Maintain running sum, count, or other metrics as the window moves
  • Expansion Strategy: Expand right pointer until condition met, then contract left pointer

⏱️ Time & Space Complexity

Complexity Type Value Explanation
Time Complexity O(n) Each element is visited at most twice (once by left pointer, once by right pointer)
Space Complexity O(k) or O(1) O(k) if using hash map for window tracking, O(1) for simple sum/count tracking

💻 Boilerplate Code

Fixed Size Window:

def fixed_window(arr, k):
    """
    Find maximum sum of subarray of size k
    """
    if len(arr) < k:
        return 0
    
    # Calculate sum of first window
    window_sum = sum(arr[:k])
    max_sum = window_sum
    
    # Slide the window
    for i in range(k, len(arr)):
        # Remove leftmost element, add rightmost element
        window_sum = window_sum - arr[i - k] + arr[i]
        max_sum = max(max_sum, window_sum)
    
    return max_sum

# TC: O(n)
# SC: O(1)

Variable Size Window:

def variable_window(arr, target):
    """
    Find minimum length subarray with sum >= target
    """
    left = 0
    window_sum = 0
    min_len = float('inf')
    
    for right in range(len(arr)):
        # Expand window
        window_sum += arr[right]
        
        # Contract window while condition is met
        while window_sum >= target:
            min_len = min(min_len, right - left + 1)
            window_sum -= arr[left]
            left += 1
    
    return min_len if min_len != float('inf') else 0

# TC: O(n)
# SC: O(1)

def longest_substring_k_distinct(s, k):
    """
    Find longest substring with at most k distinct characters
    """
    char_count = {}
    left = 0
    max_len = 0
    
    for right in range(len(s)):
        # Add character to window
        char_count[s[right]] = char_count.get(s[right], 0) + 1
        
        # Shrink window if more than k distinct chars
        while len(char_count) > k:
            char_count[s[left]] -= 1
            if char_count[s[left]] == 0:
                del char_count[s[left]]
            left += 1
        
        max_len = max(max_len, right - left + 1)
    
    return max_len

# TC: O(n)
# SC: O(k)

def min_window(s, t):
    if not s or not t:
        return ""
    
    # Count characters in t
    need = {}
    for char in t:
        need[char] = need.get(char, 0) + 1
    
    left = 0
    valid = 0  # Number of unique characters in current window matching t
    window = {}
    start = 0
    min_len = float('inf')
    
    for right in range(len(s)):
        c = s[right]
        if c in need:
            window[c] = window.get(c, 0) + 1
            if window[c] == need[c]:
                valid += 1
        
        # Try to shrink window
        while valid == len(need):
            if right - left + 1 < min_len:
                start = left
                min_len = right - left + 1
            
            d = s[left]
            if d in need:
                if window[d] == need[d]:
                    valid -= 1
                window[d] -= 1
            left += 1
    
    return "" if min_len == float('inf') else s[start:start + min_len]

Two Pointers Pattern

Two Pointers

📖 Concept

The Two Pointers pattern uses two pointers that traverse the data structure in a coordinated way. This pattern is highly efficient for solving problems involving arrays, strings, or linked lists.

Key Idea: Use two pointers moving at different speeds or from different directions to efficiently solve problems without nested loops.

🎯 Techniques to Master

  • Opposite Ends: Start pointers at opposite ends and move them toward each other
  • Fast & Slow: Use two pointers moving at different speeds (Floyd's cycle detection)
  • Same Direction: Both pointers move in the same direction at different speeds
  • Sliding Window: Two pointers define a window that slides through the array
  • Partitioning: Use pointers to partition array into sections
  • Cycle Detection: Fast pointer catches slow pointer if cycle exists

⏱️ Time & Space Complexity

Complexity Type Value Explanation
Time Complexity O(n) Each element is visited at most once by each pointer
Space Complexity O(1) Only using a constant amount of extra space for pointers

Merge Intervals Pattern

Merge Intervals

📖 Concept

The Merge Intervals pattern is used to solve problems involving overlapping intervals. The key is to sort intervals by their start time and then merge or find relationships between them.

Key Idea: Sort intervals by start time, then iterate through them, merging overlapping intervals or finding gaps/conflicts.

🎯 Techniques to Master

  • Sort First: Always sort intervals by start time (or end time for specific problems)
  • Overlap Detection: Two intervals overlap if start[i] <= end[j] and start[j] <= end[i]
  • Merge Strategy: Compare current interval with last merged interval
  • Gap Finding: Identify gaps between non-overlapping intervals
  • Conflict Detection: Find intervals that conflict with each other
  • Insert Intervals: Insert new interval and merge if necessary

⏱️ Time & Space Complexity

Complexity Type Value Explanation
Time Complexity O(n log n) Sorting takes O(n log n), merging takes O(n)
Space Complexity O(n) Space needed for storing merged intervals

💻 Boilerplate Code

Basic Merge Intervals:

def merge_intervals(intervals):
    """
    Merge overlapping intervals
    """
    if not intervals:
        return []
    
    # Sort by start time
    intervals.sort(key=lambda x: x[0])
    merged = [intervals[0]]
    
    for current in intervals[1:]:
        last = merged[-1]
        # If current overlaps with last, merge them
        if current[0] <= last[1]:
            merged[-1] = [last[0], max(last[1], current[1])]
        else:
            merged.append(current)
    
    return merged

# TC: O(n log n)
# SC: O(n)

Cyclic Sort Pattern

Cyclic Sort

📖 Concept

Cyclic Sort is an in-place sorting algorithm that works well when dealing with arrays containing numbers in a given range. It places each number at its correct index.

Key Idea: For an array of n elements with values in range [0, n-1] or [1, n], each value should be at index equal to its value. We swap elements to their correct positions.

🎯 Techniques to Master

  • Range-Based Sorting: Works best when numbers are in a known range
  • Index Mapping: Value at index i should be i (or i+1 for 1-based)
  • Swap Strategy: Swap current element to its correct position
  • Missing Number Detection: After sorting, find index where value != index
  • Duplicate Detection: Identify duplicates during sorting process
  • Boundary Checks: Ensure indices are within valid range before swapping

⏱️ Time & Space Complexity

Complexity Type Value Explanation
Time Complexity O(n) Each element is swapped at most once to its correct position
Space Complexity O(1) Sorting is done in-place, only using constant extra space

💻 Boilerplate Code

Basic Cyclic Sort (0 to n-1):

def cyclic_sort(arr):
    """
    Sort array where numbers are in range [0, n-1]
    """
    i = 0
    while i < len(arr):
        # If number is at correct position, move to next
        if arr[i] == i:
            i += 1
        else:
            # Swap to correct position
            arr[i], arr[arr[i]] = arr[arr[i]], arr[i]
    
    return arr

# TC: O(n)
# SC: O(1)

def cyclic_sort_1_to_n(arr):
    """
    Sort array where numbers are in range [1, n]

    i = 0

        correct_pos = arr[i] - 1
        if arr[i] != arr[correct_pos]:

Linked List Pattern

Linked List

📖 Concept

Linked List problems involve manipulating nodes connected via pointers. Common operations include reversal, cycle detection, finding middle node, and merging lists.

Key Idea: Use pointer manipulation to traverse, modify, or restructure linked lists without using extra space for arrays.

🎯 Techniques to Master

  • Two Pointers: Fast & slow pointers for cycle detection, finding middle
  • In-place Reversal: Reverse linked list by changing pointers
  • Dummy Head: Use dummy node to simplify edge cases
  • Pointer Manipulation: Carefully update next pointers to avoid losing references
  • Cycle Detection: Floyd's algorithm (tortoise and hare)
  • Merge Lists: Combine two sorted lists efficiently

⏱️ Time & Space Complexity

Complexity Type Value Explanation
Time Complexity O(n) Most operations require single pass through the list
Space Complexity O(1) In-place operations use only constant extra space

💻 Boilerplate Code

Reverse Linked List:

def reverse_list(head):
    """
    Reverse a linked list in-place
    """
    prev = None
    current = head
    
    while current:
        next_node = current.next
        current.next = prev
        prev = current
        current = next_node
    
    return prev

Tree BFS Pattern

Tree BFS (Breadth-First Search)

📖 Concept

BFS traverses a tree level by level, visiting all nodes at the current depth before moving to the next level. It uses a queue to process nodes.

Key Idea: Process nodes level by level using a queue. Start with root, add children to queue, process each level completely before moving to next.

🎯 Techniques to Master

  • Queue-Based: Use queue (deque) to maintain nodes at current level
  • Level Tracking: Process all nodes at current level before moving to next
  • Level Order Traversal: Return nodes grouped by level
  • Zigzag Traversal: Alternate direction at each level
  • Right Side View: Track last node at each level
  • Level Size: Track number of nodes at each level for processing

⏱️ Time & Space Complexity

Complexity Type Value Explanation
Time Complexity O(n) Visit each node exactly once
Space Complexity O(w) w is maximum width of tree (queue size at widest level)

💻 Boilerplate Code

Basic BFS:

from collections import deque

def bfs(root):
    """
    Basic BFS traversal
    """
    if not root:
        return []
    
    result = []
    queue = deque([root])
    
    while queue:
        node = queue.popleft()
        result.append(node.val)
        
        if node.left:
            queue.append(node.left)
        if node.right:
            queue.append(node.right)
    
    return result

# TC: O(n)
# SC: O(w)

def zigzag_level_order(root):
    """
    Traverse tree in zigzag pattern
    """
    if not root:
        return []
    
    result = []
    queue = deque([root])
    left_to_right = True
    
    while queue:
        level_size = len(queue)
        level = deque()
        
        for _ in range(level_size):
            node = queue.popleft()
            
            if left_to_right:
                level.append(node.val)
            else:
                level.appendleft(node.val)
            
            if node.left:
                queue.append(node.left)
            if node.right:
                queue.append(node.right)
        
        result.append(list(level))
        left_to_right = not left_to_right
    
    return result

# TC: O(n)
# SC: O(w)

Tree DFS Pattern

Tree DFS (Depth-First Search)

📖 Concept

DFS traverses a tree by going as deep as possible before backtracking. It uses recursion or a stack to explore paths.

Key Idea: Explore one path completely before backtracking. Three main orders: Pre-order (root-left-right), In-order (left-root-right), Post-order (left-right-root).

🎯 Techniques to Master

  • Recursive DFS: Use function calls to naturally traverse depth-first
  • Iterative DFS: Use stack to simulate recursion
  • Pre-order: Process root before children (root-left-right)
  • In-order: Process left, then root, then right (for BST gives sorted order)
  • Post-order: Process children before root (left-right-root)
  • Path Tracking: Maintain current path while traversing

⏱️ Time & Space Complexity

Complexity Type Value Explanation
Time Complexity O(n) Visit each node exactly once
Space Complexity O(h) h is height of tree (recursion stack or explicit stack)

Topological Sort Pattern

Topological Sort

📖 Concept

Topological Sort orders nodes in a directed acyclic graph (DAG) such that for every edge (u, v), u appears before v in the ordering.

Key Idea: Process nodes with no incoming edges first, then remove their outgoing edges and repeat. Used for dependency resolution, course scheduling, build systems.

🎯 Techniques to Master

  • Kahn's Algorithm: Use in-degree counting and queue
  • DFS-based: Use DFS with post-order processing
  • In-degree Tracking: Count incoming edges for each node
  • Cycle Detection: Detect if graph has cycles (no valid topological order)
  • Dependency Resolution: Process dependencies before dependents
  • Queue Management: Process nodes with zero in-degree

⏱️ Time & Space Complexity

Complexity Type Value Explanation
Time Complexity O(V + E) V = vertices, E = edges. Visit each node and edge once
Space Complexity O(V) Space for in-degree array, queue, and result

💻 Boilerplate Code

Kahn's Algorithm (BFS-based):

from collections import deque

def topological_sort(num_courses, prerequisites):
    """
    Kahn's algorithm using in-degree counting
    """
    # Build graph and in-degree count
    graph = [[] for _ in range(num_courses)]
    in_degree = [0] * num_courses
    
    for course, prereq in prerequisites:
        graph[prereq].append(course)
        in_degree[course] += 1
    
    # Start with nodes having no incoming edges
    queue = deque([i for i in range(num_courses) if in_degree[i] == 0])
    result = []
    
    while queue:
        node = queue.popleft()
        result.append(node)
        
        # Remove edges and update in-degrees
        for neighbor in graph[node]:
            in_degree[neighbor] -= 1
            if in_degree[neighbor] == 0:
                queue.append(neighbor)
    
    # Check if all courses were processed (no cycle)
    return result if len(result) == num_courses else []

# TC: O(V + E)
# SC: O(V)

DFS-based Topological Sort:

def topological_sort_dfs(num_courses, prerequisites):
    """
    DFS-based topological sort
    """
    graph = [[] for _ in range(num_courses)]
    for course, prereq in prerequisites:
        graph[prereq].append(course)
    
    WHITE, GRAY, BLACK = 0, 1, 2
    color = [WHITE] * num_courses
    result = []
    
    def dfs(node):
        if color[node] == GRAY:  # Cycle detected
            return False
        if color[node] == BLACK:  # Already processed
            return True
        
        color[node] = GRAY
        for neighbor in graph[node]:
            if not dfs(neighbor):
                return False
        
        color[node] = BLACK
        result.append(node)
        return True
    
    for i in range(num_courses):
        if color[i] == WHITE:
            if not dfs(i):
                return []  # Cycle found
    
    return result[::-1]  # Reverse for correct order

# TC: O(V + E)
# SC: O(V)

Union Find Pattern

Union Find (Disjoint Set Union)

📖 Concept

Union Find (Disjoint Set Union) is a data structure that tracks elements partitioned into disjoint sets. It supports efficient union and find operations.

Key Idea: Use path compression and union by rank to achieve near-constant time operations. Perfect for connectivity problems, cycle detection, and grouping elements.

🎯 Techniques to Master

  • Path Compression: Flatten tree during find operation for efficiency
  • Union by Rank: Always attach smaller tree to larger tree
  • Find Operation: Find root of element's set with path compression
  • Union Operation: Merge two sets by connecting their roots
  • Connected Check: Check if two elements are in same set
  • Count Components: Track number of disjoint sets

⏱️ Time & Space Complexity

Complexity Type Value Explanation
Time Complexity O(α(n)) α is inverse Ackermann function, effectively O(1) for practical purposes
Space Complexity O(n) Space for parent and rank arrays

💻 Boilerplate Code

Union Find with Path Compression and Union by Rank:

class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n
        self.components = n
    
    def find(self, x):
        """
        Find root with path compression
        """
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]
    
    def union(self, x, y):
        """
        Union by rank
        """
        root_x = self.find(x)
        root_y = self.find(y)
        
        if root_x == root_y:
            return False  # Already connected
        
        # Union by rank
        if self.rank[root_x] < self.rank[root_y]:
            self.parent[root_x] = root_y
        elif self.rank[root_x] > self.rank[root_y]:
            self.parent[root_y] = root_x
        else:
            self.parent[root_y] = root_x
            self.rank[root_x] += 1
        
        self.components -= 1
        return True
    
    def connected(self, x, y):
        """
        Check if two elements are connected
        """
        return self.find(x) == self.find(y)

# TC: O(α(n)) per operation
# SC: O(n)

Two Heaps Pattern

Two Heaps

📖 Concept

The Two Heaps pattern uses a min-heap and max-heap to maintain elements in two halves, allowing efficient access to median, finding smallest/largest elements, or maintaining balanced partitions.

Key Idea: Use min-heap for larger half and max-heap for smaller half. Keep heaps balanced (size difference ≤ 1) for efficient median finding.

🎯 Techniques to Master

  • Min-Heap for Larger Half: Store larger elements in min-heap
  • Max-Heap for Smaller Half: Store smaller elements in max-heap (use negative values)
  • Balance Maintenance: Keep heap sizes balanced (difference ≤ 1)
  • Median Finding: Top of heaps give median when balanced
  • Re-balancing: Move elements between heaps to maintain balance
  • Python heapq: Use heapq (min-heap) with negatives for max-heap

⏱️ Time & Space Complexity

Complexity Type Value Explanation
Time Complexity O(log n) Insert and rebalance operations take O(log n)
Space Complexity O(n) Store all elements in two heaps

            return -self.small[0]
        elif len(self.large) > len(self.small):

K-way Merge combines K sorted arrays/lists into one sorted array efficiently using a min-heap to track the smallest element from each array.

Key Idea: Use a min-heap to always get the smallest element across all K arrays. Pop smallest, add next element from same array, repeat until all arrays are exhausted.

  • Array Exhaustion: Handle when an array is fully processed
  • Divide and Conquer: Merge pairs recursively for large K

    for _ in range(k - 1):
        val, row, col = heapq.heappop(heap)
        if col + 1 < n:
            heapq.heappush(heap, (matrix[row][col + 1], row, col + 1))
    
    return heap[0][0]

def find_first_true(arr, condition):

    Find first index where condition is true

Subsets Pattern

Subsets (Backtracking)

📖 Concept

The Subsets pattern generates all possible subsets/combinations/permutations using backtracking. Each element can be included or excluded.

Key Idea: Use recursion to explore all possibilities. At each step, make a choice (include/exclude), recurse, then backtrack to try other choices.

🎯 Techniques to Master

  • Backtracking Template: Choose, explore, unchoose pattern
  • Include/Exclude: Two choices at each step
  • Duplicate Handling: Sort and skip duplicates
  • Path Tracking: Maintain current path/combination
  • Base Case: When to add current path to result
  • Pruning: Skip invalid paths early

⏱️ Time & Space Complexity

Complexity Type Value Explanation
Time Complexity O(2^n) For subsets: 2^n possible subsets (each element included/excluded)
Space Complexity O(n) Recursion stack depth and path storage

💻 Boilerplate Code

Generate All Subsets:

def subsets(nums):
    """
    Generate all subsets
    """
    result = []
    
    def backtrack(start, path):
        # Add current path to result
        result.append(path[:])
        
        # Try each remaining element
        for i in range(start, len(nums)):
            path.append(nums[i])
            backtrack(i + 1, path)
            path.pop()  # Backtrack
    
    backtrack(0, [])
    return result

# TC: O(2^n)
# SC: O(n)

Dynamic Programming Pattern

Dynamic Programming

📖 Concept

Dynamic Programming solves complex problems by breaking them into simpler subproblems and storing results to avoid recomputation. It optimizes recursive solutions with overlapping subproblems.

Key Idea: Identify optimal substructure (optimal solution contains optimal solutions to subproblems) and overlapping subproblems. Use memoization (top-down) or tabulation (bottom-up).

🎯 Techniques to Master

  • Memoization (Top-Down): Cache results of recursive calls
  • Tabulation (Bottom-Up): Build solution iteratively from base cases
  • 1D DP: dp[i] represents state at position i
  • 2D DP: dp[i][j] represents state with two dimensions
  • State Transition: Define how to compute current state from previous states
  • Space Optimization: Reduce space by reusing arrays
  • Pattern Recognition: Identify DP patterns (knapsack, LIS, edit distance, etc.)

⏱️ Time & Space Complexity

Complexity Type Value Explanation
Time Complexity O(n) to O(n²) Depends on problem. Typically O(states × transitions)
Space Complexity O(n) to O(n²) Space for DP table. Can often be optimized to O(1) or O(n)

    """

            return 0
        if remaining < 0:
            return float('inf')
        if remaining in memo:
            return memo[remaining]
        
        min_coins = float('inf')
        for coin in coins:
            result = dp(remaining - coin)
            if result != float('inf'):
                min_coins = min(min_coins, 1 + result)
        
        memo[remaining] = min_coins
        return min_coins

    return result if result != float('inf') else -1

Binary Search Tree (BST)

📖 Concept

A Binary Search Tree is a binary tree where for every node: left subtree < root < right subtree. In-order traversal yields sorted order. Used for fast search, insert, delete (average O(log n)) and range/successor queries.

Key Idea: Use the ordering property to discard one subtree per step. Validate by passing (min, max) bounds down. Kth smallest = in-order with a counter.

🎯 Techniques to Master

  • Validate BST: Pass (min, max) bounds; left must be in (min, root.val), right in (root.val, max)
  • Kth smallest: In-order traversal with counter; return when count == k
  • BST from sorted array: Mid as root, recurse left/right on halves
  • Successor / Predecessor: Use parent pointers or in-order stack

KD-Tree (K-Dimensional Tree)

📖 Concept

A KD-tree is a space-partitioning tree for organizing points in k-dimensional space. Each level splits along one dimension (cycling through dimensions). Used for nearest neighbor search, range queries, and spatial indexing.

Key Idea: Build by repeatedly splitting on median along alternating axes. Search: traverse like BST (compare on current axis), then possibly check the other subtree if the splitting hyperplane is within the search radius.

🎯 Techniques to Master

  • Build: Choose dimension (e.g. depth % k), find median, split into left/right; recurse
  • Nearest Neighbor: Go toward query point first; then backtrack to the "other side" if the hyperplane is closer than best so far
  • Range Query: Check if node's point in range; recurse left/right if the splitting plane intersects the range
  • Dimension cycling: Level 0 → axis 0, level 1 → axis 1, … level k → axis 0 again

⏱️ Time & Space (KD-Tree)

Build (n points)O(n log n)O(n²)O(n)
Nearest neighborO(log n)O(n)O(log n)

    k = len(points[0])
    axis = depth % k
    points_sorted = sorted(points, key=lambda p: p[axis])
    mid = len(points_sorted) // 2
    return KDNode(
        points_sorted[mid],
        build_kdtree(points_sorted[:mid], depth + 1),
        build_kdtree(points_sorted[mid + 1:], depth + 1)
    )

def dist_sq(p, q):
    return sum((a - b) ** 2 for a, b in zip(p, q))

def nearest(node, query, depth=0, best=None):
    if not node:
        return best
    k = len(query)
    axis = depth % k
    d = dist_sq(node.point, query)
    if best is None or d < dist_sq(best, query):
        best = node.point
    if query[axis] < node.point[axis]:
        near, far = node.left, node.right
    else:
        near, far = node.right, node.left
    best = nearest(near, query, depth + 1, best)
    # Check other side if hyperplane is within current best distance
    if far and (query[axis] - node.point[axis]) ** 2 < dist_sq(best, query):
        best = nearest(far, query, depth + 1, best)
    return best

# Build: O(n log n). Nearest: O(log n) average. SC: O(n)

📝 KD-Tree Problem

Problem: K Nearest Neighbors (KNN) with KD-Tree

Description: Given points and a query, find K nearest points. Maintain a max-heap of size K (or collect and sort) while doing nearest-neighbor-style traversal.

import heapq
def k_nearest_kdtree(root, query, k):
    heap = []  # max-heap by negative distance
    def search(node, depth=0):
        if not node:
            return
        axis = depth % len(query)
        d = dist_sq(node.point, query)
        if len(heap) < k:
            heapq.heappush(heap, (-d, node.point))
        elif d < -heap[0][0]:
            heapq.heapreplace(heap, (-d, node.point))
        if query[axis] < node.point[axis]:
            search(node.left, depth + 1)
            if len(heap) < k or (node.point[axis] - query[axis])**2 <= -heap[0][0]:
                search(node.right, depth + 1)
        else:
            search(node.right, depth + 1)
            if len(heap) < k or (query[axis] - node.point[axis])**2 <= -heap[0][0]:
                search(node.left, depth + 1)
    search(root)
    return [p for _, p in heap]

# TC: O(log n + k) average. SC: O(k + log n)

Other Spatial / Partitioning Trees

📖 Overview

R-tree: B-tree-like structure for rectangles (bounding boxes). Good for geographic data, "find all objects in this region." Insert/search by merging/splitting bounding boxes.

BSP (Binary Space Partitioning): Recursively split space with hyperplanes. Used in graphics (visibility) and collision detection.

Quadtree / Octree: Split 2D/3D space into 4/8 children at each node. Good for grid-like data, collision, LOD. Simpler than KD-tree for axis-aligned subdivisions.

When to use: KD-tree → nearest neighbor in moderate dimensions; R-tree → range queries on rectangles; Quadtree → 2D grids and games.

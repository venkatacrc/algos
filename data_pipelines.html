1. Pandas: Series & DataFrame, indexing

Core types: Series (1D, labeled) and DataFrame (2D, rows and columns). Both have an index; DataFrames also have column names.

Creation

import pandas as pd
import numpy as np

# Series from list or dict
s = pd.Series([1, 2, 3], index=['a', 'b', 'c'])
s = pd.Series({'a': 1, 'b': 2})

# DataFrame from dict of lists/arrays, or list of dicts
df = pd.DataFrame({'A': [10, 20, 30], 'B': [40, 50, 60]}, index=['x', 'y', 'z'])
df = pd.DataFrame([{'a': 1, 'b': 2}, {'a': 3, 'b': 4}])

Indexing: loc vs iloc vs at/iat

  • loc – Label-based; df.loc['x', 'B']; slices are inclusive on both ends: df.loc['x':'y'].
  • iloc – Integer position; df.iloc[0, 1]; slices are exclusive on end: df.iloc[0:2].
  • at / iat – Single scalar access (faster than loc/iloc for one cell): df.at['x', 'B'], df.iat[0, 1].
df = pd.DataFrame({'A': [10, 20, 30], 'B': [40, 50, 60]}, index=['x', 'y', 'z'])
df.loc['x', 'B']    # 40
df.loc['x':'y']     # rows x and y (inclusive)
df.iloc[0:2]        # rows 0 and 1
df.iloc[:, 1]       # column at position 1 (Series)

Index: set_index, reset_index, reindex

df.set_index('col_name')           # promote column to index
df.reset_index(drop=True)        # index → default RangeIndex; drop=True discards old index
df.reindex(['x', 'y', 'new'])    # align to new labels; missing → NaN

3. Pandas: missing data & dtypes

Missing values are represented by NaN (float). Use pd.NA in nullable integer/string types.

Detect and drop/fill

df.dropna(how='any', subset=['col1'], thresh=2)   # drop rows with < 2 non-NaN
df.fillna(0)
df.ffill(limit=2)
df.interpolate(method='linear')

Dtypes and conversion

df.dtypes
df.astype({'a': 'int32', 'b': 'category'})

5. Pandas: groupby, map, apply

GroupBy & aggregation

df.groupby('country').points.agg(['mean', 'min', 'max', 'count'])
df.groupby(['country', 'province']).price.agg([len, min, max])
df.groupby('country').agg({'points': 'mean', 'price': ['min', 'max']})
df.groupby('winery').size()   # same as .count() on any column
# Custom per-group: pick row with max points in each group
df.groupby('winery').apply(lambda g: g.loc[g.points.idxmax()])

Map vs apply vs transform

  • map – Element-wise on a Series: s.map(lambda x: x - s.mean()) or s.map({old: new, ...}).
  • apply – Row or column: df.apply(f, axis='columns') (row-wise), axis='index' (column-wise).
  • transform – Returns same shape as group; broadcast group aggregate to each row: df.groupby('g')['x'].transform('mean').
reviews.points.map(lambda p: p - reviews.points.mean())
df.groupby('region')['sales'].transform('sum')   # same-length Series

9. Pandas: I/O & chunking

Read/write

pd.read_csv('data.csv', usecols=['a', 'b'], dtype={'a': 'int32'}, nrows=1000)
pd.read_csv('data.csv', parse_dates=['date'], index_col=0)
df.to_csv('out.csv', index=False)
df.to_parquet('data.parquet', index=False)
df.to_sql('table', con=engine, if_exists='append', index=False)

Chunked read for large files

chunks = pd.read_csv('huge.csv', chunksize=100_000)
results = []
for chunk in chunks:
    results.append(chunk.groupby('region').size())
summary = pd.concat(results).groupby(level=0).sum()

Pandas scale limits

  • Single machine, single process; data should fit in RAM (or use chunks).
  • For larger data, use PySpark, Dask, or Polars.

10. PySpark: session, read, lazy evaluation

PySpark distributes work across a cluster. Operations are lazy: transformations build a DAG; actions (collect, count, write) trigger execution.

Session & reading data

from pyspark.sql import SparkSession

spark = SparkSession.builder.appName("pipeline").getOrCreate()
df = spark.read.csv("s3://bucket/data/", header=True, inferSchema=True)
df = spark.read.parquet("s3://bucket/data/")
df = spark.read.json("s3://bucket/events/")

Transformations (lazy) vs actions

  • Transformations – select, filter, withColumn, drop, groupBy, join, union, repartition, orderBy. Return a new DataFrame; no work yet.
  • Actions – count, collect, show, take, write, foreach. Trigger the full plan.
from pyspark.sql.functions import year, sum
out = (
    df.filter(df.amount > 0)
      .withColumn("year", year(df.date))
      .groupBy("year", "region")
      .agg(sum("amount").alias("total"))
)
out.write.mode("overwrite").parquet("s3://bucket/out/")  # action

Column references and literals

df.withColumn("safe", coalesce(col("a"), col("b"), lit(0)))

13. PySpark: joins, union, dedup

Joins

df1.join(df2, df1.id == df2.id, "inner")   # inner, left, right, full, cross, semi, anti
df1.join(df2, "id")   # natural join on column "id"
df1.join(df2, [df1.a == df2.a, df1.b == df2.b])

from pyspark.sql.functions import broadcast
df_large.join(broadcast(df_small), "key")

Union and deduplication

df1.union(df2)           # no dedup (like SQL UNION ALL)
df1.union(df2).distinct()
df1.unionByName(df2)    # match by name; allow missing columns with allowMissingColumns=True
df.dropDuplicates(["col1", "col2"])

14. PySpark: window functions & SQL

Window functions

from pyspark.sql import Window
from pyspark.sql.functions import row_number, rank, dense_rank, lag, lead, sum as spark_sum

w = Window.partitionBy("user_id").orderBy(col("ts"))
w_unbounded = Window.partitionBy("user_id").orderBy("ts").rowsBetween(Window.unboundedPreceding, Window.currentRow)

df.withColumn("row_num", row_number().over(w))
df.withColumn("prev_amt", lag("amount", 1).over(w))
df.withColumn("running_total", spark_sum("amount").over(w_unbounded))

SQL

df.createOrReplaceTempView("events")
spark.sql("SELECT region, SUM(amount) FROM events GROUP BY region").show()
spark.sql("SELECT *, ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY ts) AS rn FROM events")

15. PySpark: UDFs & Pandas UDF

Row-at-a-time UDF (slower, serialization overhead)

from pyspark.sql.functions import udf
from pyspark.sql.types import StringType

@udf(returnType=StringType())
def my_upper(s):
    return s.upper() if s else None

df.withColumn("name_upper", my_upper(col("name")))

Pandas UDF (vectorized, faster)

from pyspark.sql.functions import pandas_udf
import pandas as pd

@pandas_udf("double")
def mean_udf(s: pd.Series) -> pd.Series:
    return (s - s.mean())

df.withColumn("centered", mean_udf(col("x")))

16. PySpark: cache & persistence levels

Cache and persist

df.cache()   # same as persist(StorageLevel.MEMORY_AND_DISK)

from pyspark import StorageLevel
df.persist(StorageLevel.MEMORY_AND_DISK)
df.persist(StorageLevel.DISK_ONLY)
df.persist(StorageLevel.MEMORY_AND_DISK_SER)   # serialized (less memory, more CPU)
df.unpersist()

Use cache() or persist() when a DataFrame is reused across multiple actions. First action materializes the cache; later actions reuse it.

18. Distributed data optimizations for large datasets

When data is large or distributed, optimize for I/O, shuffles, and parallelism.

Partitioning

  • Input partitioning – Store data already partitioned by a key (e.g. date=2024-01-01/, country=US/). Reads can skip irrelevant partitions (partition pruning).
  • Spark repartition vs coalesce – repartition(n) shuffles into n partitions (use before heavy aggregation or to increase parallelism). coalesce(n) merges to n without full shuffle (use to reduce partitions before write).
# Write partitioned by column
df.write.partitionBy("date", "region").parquet("s3://bucket/out/")

# Read only some partitions
spark.read.parquet("s3://bucket/out/").filter("date = '2024-01-01'")  # partition pruning

# Repartition before wide transformation; coalesce before single-file write
df.repartition(200, "user_id").groupBy("user_id").sum("amount")
df.coalesce(1).write.csv("single_file.csv")  # fewer output files

File formats

  • Parquet – Columnar; compression; predicate pushdown (skip row groups); good for analytics. Default choice in Spark.
  • ORC – Also columnar; similar benefits; used in Hive.
  • CSV – Row-based; no pushdown; avoid for very large distributed data.

Predicate pushdown

Filters are pushed to the reader: only row groups (Parquet) or splits that can satisfy the filter are read. Use explicit filters: df.filter("date = '2024-01-01'") so the engine can push them down.

Broadcast join

When one side of a join is small, broadcast it to every executor so the join is local (no shuffle).

# Spark can auto-broadcast under spark.sql.autoBroadcastJoinThreshold
df_large.join(broadcast(df_small), "key")

Caching

Reused DataFrames: df.cache() or df.persist(storageLevel). Data is kept in memory (or memory+disk) after the first action so subsequent actions don’t recompute. Use for iterative algorithms or when the same DF is read multiple times.

base = spark.read.parquet("s3://bucket/events/").filter("year = 2024").cache()
base.count()  # materializes cache
base.groupBy("region").count().show()
base.join(other, "id").count()  # reuses cached base

Pandas on large data (single machine)

  • Chunked reads – pd.read_csv(..., chunksize=N); process and aggregate per chunk.
  • Memory-mapped files – np.memmap or columnar formats so the OS pages in only what’s needed.
  • Dask – Pandas-like API; runs on a single machine or cluster; lazy and partitioned. Alternative when you want “Pandas but bigger” without full Spark.

Summary: optimization checklist

  • Store data partitioned by common filter/join keys; use columnar formats (Parquet/ORC).
  • Apply filters early so predicate pushdown and partition pruning apply.
  • Broadcast small tables in joins; repartition for even parallelism; coalesce to limit output files.
  • Cache DataFrames that are reused; avoid unnecessary collect() of large results.
